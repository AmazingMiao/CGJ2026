# 《西西弗斯和他的遥控巨石》· 概念大纲

> CIGA Game Jam · 主题 **Anchor** · Unity 2D · 5 人(2 程序 + 2 美术 + 1 策划)· 48h
> 本文档用于全队对齐当前已确定的核心概念,并标出还没拍板、需要尽快定的点。

---

## 一句话概念

一个**单人双设备**平台跳跃游戏:PC 键盘控制西西弗斯角色跳台,手机陀螺仪隔空控制一块巨石的**重力方向**。巨石既是会把你压死的凶器,也是把你轰上山的跳板——你得带着这块甩不掉的石头一起爬到终点。

**梗与主题合一**:西西弗斯不推石头了,他搞了个遥控器。手机 = 遥控器 + 监控屏,巨石 = 那块永恒的诅咒之石(也就是"锚")。

---

## 核心机制(已锁定)

- **两个独立刚体,无铁链**。角色和巨石各自自由,不再互相拴着。
- **PC 键盘**:控制角色本体(左右移动、跳)。
- **手机陀螺仪**:只控制**巨石受到的重力方向**。倾斜手机 → 石头往那个方向滚/砸/坠。
- **巨石是移动工具 + 凶器**:
  - 高速横向撞到玩家 → 把玩家**撞飞**。
  - 玩家撞墙 → **反弹**(墙面反弹是核心移动手段)。
  - 玩法核心动词:主动让石头以合适的角度和速度撞自己,用它当发射器把自己送到够不到的地方。

### 生死判定(已锁定)

以「玩家背后有没有退路」决定生死,规则简单、玩家一学就会,程序检测撞击反方向有无碰撞体即可:

| 情况 | 结果 |
|---|---|
| 玩家背后是**空的** + 石头撞过来 | **撞飞**(正常玩法) |
| 玩家背后有**墙/地** + 石头压过来(两面夹击) | **压死 → 死亡** |

### 检查点与推进(已锁定)

- **巨石和玩家必须一起到检查点**,进度才算数(存档条件)。逼玩家带着石头走,不能甩掉石头自己跑。
- **被压死** → 回**当前检查点**重来。
- **玩家直接掉进下一关区域** → 直接进入**那一关的检查点**。

### 石头追踪(已锁定)

- 石头飞出主视野时,**手机屏幕上显示它的追踪画面**——手机既是控制器又是监控屏,符合"遥控器"直觉。
- **不是视频推流**,而是**极简示意图/雷达**:用简单图形表示石头、玩家、墙的相对位置。PC 只回传少量坐标数据,手机本地用 canvas 画。数据量小、更省事,粗糙的雷达小屏也更 meme、更对味。
- 效果上更好:操控石头和看石头都在手机上,眼睛跟着手走,不用在两块大屏之间分裂注意力。

## Anchor 主题怎么扣

- 巨石**字面上就是锚**,而且天然双重性:它是死重、是拖累(束缚),同时又是唯一的过关工具(依靠)。
- 西西弗斯本身 = 被永恒锚定在命运上的人。"我被这块石头困了一辈子,现在我要用它爬出去"——束缚与救赎的张力被套皮直接拉满。
- 手机物理倾斜去"锚定"重力方向,把"锚"从比喻拉回物理动作,扣题很实。

---

## 美术方向(已定基调)

- **希腊肌肉大叔**,风格越 meme、越糙越对味,**别往精致写实走**。
- **纵向攀爬**场景——"推石上山"和平台跳跃的"往上爬"天然重合。
- 手机端:一个粗糙的"监控/雷达小屏"边框。
- 美术量先控制在:角色 + 巨石 + 一套 tile + 手机 UI 边框。别一开始就想做多套皮肤或复杂背景。

## 结局钩子(建议保留,几乎零成本)

玩家历尽千辛把巨石送到山顶,它**稳稳停住没滚下来**,西西弗斯愣一下,坐下开始玩手机。一个静态画面 + 一行字,做 demo 记忆点的性价比极高。

## ⚠️ 还没定 / 需要尽快拍板

**1. 玩家掉到下一关时,石头怎么办?**(必须定,否则会出现"玩家在下层、石头卡在上层"的死局)
- (a) 玩家进新区域 → 石头强制一起拽过来。*(推荐:省心、不卡关)*
- (b) 换区域后,得等石头也下来才真正存档。
- **→ 需要拍板。**

**2. 石头进入拿不回来的死局(掉虚空 / 卡死角)要不要软兜底?**
- 手机雷达负责"够得着的远";建议再加一个保险丝:石头彻底出界超过 X 秒 → 自动送回玩家附近或当前检查点。
- **→ 需要确认要不要做、阈值多少。**

**3. 核心手感数值(策划 tune)**:撞飞力度、反弹衰减系数、多快算"高速撞击"。
- **→ 等原型出来后专门调,先留位。**

**4. 纵向关卡机制清单**:围绕核心动词(撞飞跨深渊 / 躲夹击 / 石头反弹连跳 / 看第二屏)拆 5–8 个短关,**一关只强调一种用法**,别全堆一起。
- **→ 待策划展开。**

## 技术可行性 & 风险(按优先级)

**🔴 头号风险 —— 手机↔PC 双向通信,第一天上午必须先验通。**
常见做法:手机开网页(浏览器 `DeviceOrientation` API),通过局域网 WebSocket/UDP 与 PC 端 Unity 通信。
- 手机 → PC:发陀螺仪倾角。
- PC → 手机:回传石头/玩家坐标(供雷达显示)。
- **最小验证目标**:①手机一歪,PC 上一个方块就动;②手机能同时收到 PC 发来的少量坐标并画个简单界面。
- 通了再往下做;通不过立刻切 Plan B。**这块很容易吃掉一个程序一整天,越早验越好。**

**🟡 Plan B(通信搞不定的兜底)**:巨石的重力方向改用**鼠标 / 手柄右摇杆**控制,全程 PC 单机。玩法内核不变,只丢掉"物理倾斜"这层扣题。提前想好,随时能切。

**🟢 物理**:两个 `Rigidbody2D` + 碰撞检测,无铁链、不调链子,比想象中简单。主要变量集中在撞飞力度和反弹系数的数值调校,可控。

**关卡设计提醒**:巨石现在同时是武器 / 跳板 / 会压死你的东西,身份多。关卡务必**一关一种用法**教学,否则玩家会被这块万能又致命的石头搞懵。
